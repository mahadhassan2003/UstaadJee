# UstaadJee: AI Service Orchestrator for Informal Economy

**UstaadJee** is an Agentic AI system that automates the end-to-end lifecycle of a service request in Pakistan's informal economy. From natural language intent parsing (Urdu, Roman Urdu, English) to provider matching, booking simulation, and follow-up scheduling, UstaadJee transforms the chaotic process of finding reliable technicians, plumbers, and tutors into a seamless, autonomous workflow.

## 🚀 Built with Google Antigravity

This entire project was architected, coded, and tested using **Google Antigravity** as the core development platform.

### Backend Stack
*   **Framework:** FastAPI (Python)
*   **Agent Pipeline:** LangGraph by LangChain (5-node graph)
*   **LLM:** Groq (Llama 3.3 70B Versatile)
*   **State & Memory:** LangGraph SQLite Checkpointer (multi-turn memory)
*   **Real-time Streaming:** Server-Sent Events (SSE)

### Frontend Stack
*   **Framework:** React Native (Expo Web & Mobile)
*   **State Persistence:** SQLite-backed persistent chat history API
*   **Design:** Custom, premium UI with real-time step-by-step animations

## 🤖 The Agentic Workflow (5-Node Pipeline)

When a user types: *"Mujhe kal subah G-13 mein AC technician chahiye"*

| Node | Agent | What it does |
|------|-------|-------------|
| 1 | `extract_intent` | LLM processes multilingual input → extracts service, location, time. If incomplete, asks a clarification question |
| 2 | `find_providers` | Queries mock database for providers matching the service type and city |
| 3 | `rank_recommend` | Calculates Haversine distance, factors in rating/reviews/experience → ranks top 3. LLM generates reasoning for #1 pick |
| 4 | `simulate_booking` | Creates booking ID, assigns time slot, writes to SQLite database, generates confirmation receipt |
| 5 | `schedule_followup` | Creates 4 automated reminders (day before, 1hr before for user & provider, completion check) |

## ⚠️ Assumptions & Limitations
*   **Location Data:** We simulate coordinates using the Haversine formula with predefined sector locations (G-13, F-8, etc.) instead of paid Google Maps APIs.
*   **Mock Providers:** 20 simulated providers with realistic but fictional data — no real PII used.
*   **Booking Simulation:** Instant confirmation. In production, this would trigger FCM push notifications with a 60-second provider acceptance window.
*   **Payment:** Not implemented. Would integrate JazzCash/EasyPaisa QR codes in production.
